## Activity Instructions

In this activity, you are practicing how to declare the "Header" or "Signature" of a method. You are acting as the architect, not the builder. 

Open the `MethodDefinition.java` file and read the 20 descriptions. For each description, write **only the method signature** as a comment. 

**What is a Method Signature?**
It is the very first line of a method that defines its access level, return type, name, and parameters. 
Example: `public static int calculateSum(int x, int y, int z) {}`

> [!WARNING]
> ### 👾 Break-Proofing Your Code (Adversarial Thinking):
> * **The Implementation Trap:** DO NOT WRITE THE LOGIC FOR THESE METHODS! If a prompt asks for a method that reads a file or prints the Fibonacci sequence, do not actually write the code to do that. You are *only* writing the header and the empty curly braces `{}`. 
> * **Access Modifiers:** Pay close attention to the words in the prompt. If it says "A private method" or "A protected method," you must change the encapsulation type from `public` to `private` or `protected`.
> * **Return Types vs. Printing:** If a prompt says "returns the average", the return type must be a data type (like `double`). If the prompt says "prints all the characters", the method isn't returning anything, so the return type must be `void`!

### Expected Output
